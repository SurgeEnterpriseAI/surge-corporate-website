AI Agents & Model Context Protocol (MCP) for Enterprise

Connect AI agents to your enterprise data securely, with the open standard built for it.

Home › AI & Innovation › AI Agents & Model Context Protocol (MCP) for Enterprise

The Model Context Protocol (MCP) is the open standard for connecting AI agents to enterprise data and tools. Originally developed by Anthropic, MCP is now governed by the Linux Foundation's Agentic AI Foundation and has passed 110 million monthly SDK downloads. It lets AI agents safely access trusted data systems without custom-built APIs. Surge, a Premier Adobe Experience Cloud Partner, helps enterprises adopt, integrate, and govern MCP-based AI agents.

What MCP is and why it matters for enterprises

MCP is a shared 'language' that lets AI agents talk to your business systems in a predictable, inspectable way. Instead of building rigid custom APIs for every connection, you use one open standard. That makes it easier to move from experimental copilots to real operational use cases, and to run multiple agents together (called multi-agent orchestration) safely.

  • Open standard: donated to the Agentic AI Foundation under the Linux Foundation in December 2025, with support from OpenAI, Google DeepMind, and Microsoft.
  • Enterprise-ready security: MCP 2.0 added OAuth-based authorization, structured schemas, elicitation flows, and blast-radius controls for safe autonomous agents.
  • Built to scale: the 2026 roadmap moves to a stateless transport design that works correctly behind load balancers and proxies.
  • Less context bloat: new 'Skills Over MCP' and 'Progressive Discovery' features let agents load only the skills and context a task needs.
  • Native adoption: platforms like Salesforce, Informatica, and Snowflake are integrating MCP so agents can reach trusted data without rigid custom APIs.

Frequently Asked Questions

What is the Model Context Protocol (MCP) for enterprises?

MCP is an open standard that connects AI agents to enterprise data systems and tools. It provides a safe, inspectable boundary so agents can access trusted data predictably, without custom-built APIs. It's now governed by the Linux Foundation's Agentic AI Foundation.

How does MCP 2.0 improve security?

MCP 2.0 was released to close enterprise security gaps. It adds OAuth-based authorization, structured schemas, elicitation flows, and blast-radius controls, shifting the focus from basic enablement to risk mitigation for safe autonomous agent deployments.

Which platforms support MCP?

Major enterprise data platforms are adopting MCP natively, including Salesforce, Informatica, and Snowflake. These integrations let specialized AI agents securely access trusted data silos and perform agentic orchestration without rigid custom APIs.

Can MCP scale for large enterprise deployments?

Yes. The 2026 roadmap prioritizes a stateless transport redesign that behaves correctly behind load balancers and proxies, without requiring live connections for discovery. New 'Skills Over MCP' and 'Progressive Discovery' features also reduce context bloat at scale.
